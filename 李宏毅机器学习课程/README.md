同一条数据(特征维度为 $d$)，理论推导时用列向量表示，实际用编程时是用行向量表示的，这是因为：

- 推导时，线性代数中向量默认为列向量，如线性层表达式为 $\mathbf{y} = W\mathbf{x} + \mathbf{b}$ (权重矩阵 $W$ 左乘)，求导和链式法则推导更顺畅。

- 编程时， $N$ 条数据堆叠成 $N \times d$ 矩阵，这种“行对应样本”，“列对应特征”的结构与表格数据完全一致，符合直觉。同时，底层硬件(CPU/GPU)通常按“行优先”存储连续内存。把一条数据存在同一行，读取和计算的效率最高。

从公式到编程，由于数据形状发生转置(列变行，行变列)，所以所有的推导也要做转置：

- 例如 $(W\mathbf{x})^T = \mathbf{x}^T W^T$ ，$\mathbf{x}$ 是列向量
- PyTorch 实际计算时，例如 `nn.Linear` 内部执行的是 $Y = X W^T + \mathbf{b}$ ，输入 $X$ 是 $N \times d$ 的批次，经过计算直接得到 $N \times k$ 的输出，保持“行对应样本”，“列对应特征”的结构不变。

总之，推导时一条数据是列向量，编程时一条数据是行向量。
