# 为什么需要可解释的机器学习模型

我们需要机器给出得到某一个答案的理由，例如：

- 银行用机器学习模型来做是否放贷的判断，必须要给出放贷与不放贷的理由。

- 机器学习用在医疗诊断上，必须给出诊断的理由。

- 机器学习模型帮助法官判案，并给出判决的理由。

- 自动驾驶突然急刹时，需要了解它急刹的理由。

后续我们可以根据机器给出的理由去修正模型。

可解释的机器学习模型的目标就是给予人一个能够接受的理由。

# 可解释性 vs 功能性

不使用深度学习的模型，改采用其他比较容易解释的模型，比如线性模型。根据一个线性模型中每一个输入特征的权重，可以知道线性模型在做什么事。虽然比较容易解释，但线性模型功能不强大，有很多限制。

决策树相较于线性模型是更强大的模型，且相较于深度学习，决策树的可解释性更强。但只使用决策树的模型可能不够强大，一般都是使用数棵决策树，也就是随机森林，但是使用随机森林又会导致可解释性下降。

# Local Explanation & Global Explanation

Local Explanation：根据某一个输入样本进行回答。例如：给模型一张图片，模型判断是一只猫，问模型为什么觉得这张图片是一只猫。

Global Explanation：根据模型参数本身分析原因。例如：不给模型任何特定图片，需要解释对具有一堆参数的模型而言，什么样的东西叫作一只猫。

## Local Explanation

### Which component is critical ？

component 可以是图片的像素、文章的词汇等等，对每一个 component 做变化、或删除，如果模型的输出有了巨大的变化，就表示该 component 很重要。例如：对于图像分类模型，在图片不同的位置放上灰色的方块，当这个方块遮挡住图片的某一个关键区域时，模型会给出不同的分类结果。

### Saliency Map

Saliency Map是一种专门用于视觉模型的解释技术。

每一个 $x_i$﻿ 代表图片一个像素，$e$ 是模型输出的结果跟正确答案的交叉熵损失。 要考察某一个像素的重要性，可将该像素改变 $\Delta x_i$ ﻿，此时 $e$﻿ 也会发生改变 $\Delta e$ 。比值 $\displaystyle\frac{\Delta e}{\Delta x_i}$ 表示该像素发生改变时，对图片识别结果的影响，也就是该像素的重要性。

$e$ 对每一像素 $x_i$ 的偏导 $\displaystyle\frac{\partial e}{\partial x_i}$ 都算出来就得到 Saliency Map 。

Saliency Map 的改进：Smooth Grad。在某一图片上面加上各种不同的噪声，得到各种不同的图片后，对每一张图片计算 Saliency Map，平均起来得到 Smooth Grad 的结果，这样得到的结果往往能够更加集中在被侦测的物体上。

### 模型如何处理输入数据

1. 可视化分析：将高维、抽象的模型内部数据转化为人类视觉可理解的图像或图表：

- 降维可视化： 深度学习模型通过逐层提取特征，将原始数据映射到高维空间。可视化工具，如 t-SNE, PCA, UMAP 可以将这些成百上千维的向量降维到 2D 或 3D 空间中并画成散点图。
- 热力图：在自然语言处理和计算机视觉中，可以直接将模型的注意力权重或梯度映射回原始输入上。例如：将模型关注的像素区域高亮显示，通过连线或颜色深浅，展示模型在生成某个词时，注意力集中在输入句子的哪些单词上。

2. Probing：探测某一层是否学到了东西，其步骤是：

- 拿一个训练好的大模型，固定其所有参数，不再更新。
- 将数据输入模型，截取其中间某一层的隐藏层输出。
- 将截取出的隐藏层输出送入一个极其简单的分类器(通常是线性回归或单层神经网络)来进行分类任务(如判断词性)。如果能够在分类任务上取得很高的准确率，就认为：被测试的那一层中，模型的学习效果比较好。但有时很难界定是模型的学习效果比较好，还是探针分类器自己“学”到了解决任务的能力(因此探针模型必须设计得足够简单)。

## Global Explanation

假设我们有一个训练好的图像分类模型(CNN)， $y_i(X)$ 表示输入图像 $X$ 属于第 $i$ 类的概率。

要找到模型认为的第 $i$ 类图像，最直观的做法就是无中生有找一张图像 $X^*$，让模型判定它是第 $i$ 类的概率最高。这可以写成一个优化问题：

$$
X^* = \arg\max_X y_i(X).
$$



固定模型的参数，把输入图像 $X$ 当作需要学习的“参数”。通过梯度上升(Gradient Ascent)的方法，不断调整 $X$ 的像素值，使得 $y_i(X)$ 越来越大。直接用上述公式找出来的 $X^*$ 大概率是一堆噪声，但模型看到这堆噪声，却会给出 99.9% 的置信度认为这是第 $i$ 类的图像。这也解释了为什么在[对抗性攻击](./08-对抗性攻击(Adversarial Attack).md)中，只需要一些噪声就能成功让模型误判。

### 增加额外约束

为了让模型生成的图像不仅属于第 $i$ 类的概率高，还要像一张正常图片，所以需要加入人类的先验知识作为约束，这就引入了额外的约束项 $R(X)$ 用来评估图像 $X$ 有多像一张真实的图片：

$$
X^* = \arg\max_X (y_i(X) + \lambda R(X)).
$$

常见的 $R(X)$ 设计方法包括：

- 限制像素值不要太大，或者希望大部分像素是白色的，只有少数笔画。
- 限制相邻像素之间的差异不要太大(让图像变得更平滑，而不是充满了高频噪点)。

加入了 $R(X)$ 之后，生成的图像会好一些，但往往还是一张无法让人满意的诡异图像。

### 利用 Generator

借用 GAN 中的生成器 $G$ ，输入一个低维的随机向量 $z$，输出一张逼真的图片 $X = G(z)$ ，只要是 $G$ 生成的图片，看起来就一定像真实世界的图片，所以不再直接优化图像的像素 $X$，而是去优化那个低维向量 $z$ ：

$$
z^* = \arg\max_z y_i(G(z)).
$$
